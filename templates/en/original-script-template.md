# Original Script Template

This file is a production template.  
It is not a workflow. It is not a theory guide.

Use it after the character concept, setting logic, relationship frame, and behavioral structure are already defined.

The goal is to turn an original RP bot concept into a stable final script structure built for long-form play, progression, and resistance to generic model drift.

Not every block is mandatory. Use only the blocks that are structurally relevant to the bot.

This template is modular.
Do not assume that relational, romantic, domestic, or intimacy-oriented sections are automatically the center of every bot.

If the bot is primarily driven by another structure — such as combat, politics, leadership, religion, investigation, survival, professional duty, criminal logic, family hierarchy, magical systems, or another specialized axis — create dedicated specialized modules for that axis and place them where they belong in the bot’s actual behavioral hierarchy.

A specialized module should be placed before relational or intimacy-oriented sections whenever it governs:
- the character’s decisions
- scene generation
- conflict behavior
- long-term continuity
- or the overall shape of the bot more strongly than those sections do

If you are unsure how to prioritize the script, you can use an AI assistant such as ChatGPT, Claude, or Gemini to help audit the bot’s actual structural drivers.

That means asking the model to identify:
- which blocks are truly central to the bot
- which modules are secondary
- which structures are driving scenes, conflict, and long-term continuity most strongly
- which modules should be merged, separated, or moved earlier in the final script

The model should not decide the bot for you blindly.
It should be used as a structural analysis tool to help determine the bot’s real hierarchy rather than defaulting to a fixed relational template.

The template should reflect the bot’s real structure, not force every original character into the same order of importance.

## [SETTING]

Write the character’s current environment and life context.

Include only what actively shapes behavior in RP:
- living situation
- work or occupation
- social environment
- material conditions
- local routines
- risks, instability, obligations, or pressures
- any relevant world rules or setting constraints

This block should explain where the character exists and what kind of life structure the bot is operating inside.

---

## [CORE CONCEPT]

Write the core premise of the bot in playable terms.

Answer questions like:
- Who is this character in the world?
- What kind of dynamic is this bot built to sustain?
- What makes this character structurally interesting in RP?
- What pressure, contradiction, role, or emotional logic makes them playable over time?

This is not a tagline.  
It should define the bot’s central function and identity as a roleplay partner.

---

## [FOUNDATION]

Write the character’s baseline life structure at the start of RP.

State:
- what their life currently looks like
- what pressures define it
- what emotional or practical conditions shape daily behavior
- what remains stable at the start of play
- what is already broken, tense, missing, avoided, desired, or unsustainable

This block should create the starting baseline the rest of the script builds from.

---

## [CORE IDENTITY]

Write the behavioral core of the character as it should appear in RP.

Focus on:
- decision logic
- priorities
- emotional expression style
- restraint patterns
- pressure response
- recurring habits of thought or action
- what the character does when something matters
- what they protect, avoid, chase, suppress, or return to

This block should explain how the character behaves, not how they would be marketed in a profile.

Avoid relying on personality labels alone.  
Use concrete behavioral logic whenever possible.

---

## [BEHAVIORAL SIGNATURE]

Write the recurring behavioral texture of the bot in scenes.

Focus on what the character tends to:
- notice
- initiate
- conceal
- avoid
- test
- tolerate
- interrupt
- push
- physically do in space
- return to under stress, attraction, anger, boredom, or uncertainty

This block should capture how the bot *feels to roleplay against* scene by scene.

It should help distinguish the character from generic archetypes with similar themes.

---

## [SPECIALIZED MODULE]

Optional. Repeat as needed.

Use this block when a specialized structure is too behaviorally important to leave implicit, but too specific to fit cleanly inside the shared core blocks.

Unlike a minor add-on, a specialized module can be one of the main structural centers of the bot.
If it governs behavior more strongly than the relational blocks, place it before them.

Examples:
- combat structure
- political function, diplomacy, governance, or power management
- military conditioning or command hierarchy
- criminal network logic or underworld structure
- religious practice, ritual structure, taboo, or faith-based obligation
- magical rules that directly shape behavior
- chronic illness management or disability-related behavioral structure
- public career, fame, media pressure, or reputation management
- parenthood, household authority, or family hierarchy
- species-specific instincts or supernatural drives
- addiction structure
- debt, coercion, or dependency structure
- research, investigation, or operational logic
- any other system that remains behaviorally active and repeatedly shapes scenes

For each specialized module, define:
- what it is
- how it shapes decisions
- how it shapes scene behavior
- how it affects conflict, attachment, restraint, escalation, or vulnerability if relevant
- what continuity it should preserve across scenes

Do not use this block as a miscellaneous overflow section.
If the content fits a core block, place it there instead.

---

## [ROLE IN USER'S LIFE]

Define the user’s role in the bot’s life and the starting relationship frame.

State:
- what already exists between them
- what does not exist yet
- what the character assumes about the user
- what kind of access, tension, familiarity, distance, dependence, or obligation already exists
- what kind of progression the setup is designed to allow

This block should anchor the relationship before the RP begins.

---

## [RELATIONAL MODEL]

Optional if relational behavior is structurally important to the bot, or if the relationship with the user is one of the main engines of play.

Describe how the character handles closeness and interpersonal attachment in practice.

Focus on behavior around:
- trust
- suspicion
- dependency
- protectiveness
- resentment
- vulnerability
- jealousy
- emotional withholding
- emotional escalation
- forgiveness
- possessiveness
- distance vs pursuit
- comfort vs control

This block should explain how the character behaves in relationships, not just what they “feel deep down.”

---

## [SCENE ENGINE]

State what naturally generates scenes with this bot.

Focus on the recurring sources of movement such as:
- duties
- needs
- temptations
- habits
- emotional contradictions
- external threats
- household dynamics
- rivalry
- dependence
- work obligations
- survival pressure
- social pressure
- family pressure
- secrecy
- appetite
- boredom
- loneliness
- desire
- territoriality
- guilt
- curiosity
- specialized obligations or structures tied to the bot’s world

This block is one of the most important parts of an original bot script.

It should answer:
**Why does interaction keep happening with this character beyond surface attraction or small talk?**

If the bot is mainly driven by a non-relational axis, that should be clearly visible here.

---

## [WORLD ANCHORS]

Write the parts of the world that need to remain behaviorally active in RP.

Examples:
- job
- rank or status
- household structure
- financial instability
- danger
- family expectations
- physical isolation
- public reputation
- surveillance
- class position
- social obligations
- neighborhood or community ties
- routines, recurring spaces, recurring people

This block prevents the character from floating in an empty void once the RP begins.

---

## [ENVIRONMENT / HOME / DOMESTIC LIFE]

Optional if relevant.

Describe how the character inhabits private space and routine life.

Focus on things like:
- comfort habits
- territoriality
- hospitality
- cleanliness or disorder
- food habits
- sleeping habits
- silence
- noise
- shared-space behavior
- private rituals
- nesting, hoarding, maintenance, decoration, control, neglect

This block is especially useful for bots built around intimacy, cohabitation, domestic tension, caretaking, possessiveness, or routine proximity.

---

## [CONTINUITY ANCHORS]

State what must remain behaviorally continuous once established in RP.

Include things like:
- resolved conflicts that should stay resolved
- changes in trust
- attachment progression
- recurring comforts or rituals
- household changes
- new boundaries
- promises
- admissions
- jealousy triggers
- routines that become shared
- consequences of intimacy, betrayal, fear, or reassurance
- specialized pressures or systems that should remain active once they enter play
- world consequences that should continue shaping later scenes

This block protects long-form continuity and prevents the bot from resetting emotionally or relationally between scenes.

---

## [ATTACHMENT]

Optional if attachment progression is structurally important to the bot.

Describe:
- what increases attachment
- what threatens it
- how the character shows attachment
- what attachment changes in behavior once present
- how attachment interacts with pride, fear, control, shame, protectiveness, need, resentment, or avoidance

Focus on progression and visible consequences rather than abstract declarations of feeling.

---

## [CONFLICT]

Describe how the character handles conflict.

Focus on:
- anger
- hurt
- blame
- pride
- guilt
- silence
- confrontation
- escalation
- apology
- repair
- withdrawal
- punishment
- emotional retaliation
- defensiveness
- unresolved tension

This block should make it clear what conflict *looks like* with this character and how conflict changes over time.

If specialized modules materially affect conflict behavior, make sure the logic stays aligned with them.

---

## [ROMANCE]

Optional if romance is structurally relevant.

Describe:
- attraction style
- pacing
- romantic barriers
- emotional thresholds
- confession patterns
- fear around romance
- what romance changes in behavior
- what the character does with tenderness, longing, exclusivity, or vulnerability

Use this block only if romance is a real part of the bot’s intended design.

---

## [INTIMACY]

Optional if sexual or physical intimacy is structurally relevant.

Describe:
- initiative style
- hesitation or restraint
- confidence vs awkwardness
- control, patience, urgency, teasing, greed, softness, ritual, avoidance
- touch habits
- comfort boundaries
- consent assumptions to avoid
- aftercare or post-intimacy behavior
- emotional consequences of sex or physical closeness

This block should help prevent generic intimacy writing and keep the character’s physical behavior specific to them.

---

## [PREGNANCY]

Optional.

Use only if pregnancy, breeding, fertility anxiety, parenthood, reproductive pressure, or family-building themes are structurally relevant to the bot.

Do not include this block by default.

---

## [MOMENTUM]

Describe what keeps scenes moving.

State how the bot should create progression instead of replacing movement with:
- repeated questions
- hesitation loops
- passive observation
- emotional resets
- endless clarification
- static yearning
- generic flirting without escalation or consequence
- attraction that never changes behavior
- conflict that never reaches a new state
- vulnerability that never produces decisions

This block should define how the bot participates in forward motion.

Make sure momentum reflects the bot’s real structure.
If combat, politics, survival, investigation, leadership, work logic, criminal logic, family hierarchy, magical systems, or another specialized axis drives scenes more strongly than relational progression does, that should be visible here.

---

## [ANTI-GENERIC]

List the generic archetypal versions of this bot that the model must not collapse into.

Examples:
- generic cold mysterious man
- generic dominant possessive lover
- generic snarky flirt
- generic damaged sweetheart
- generic soft domestic partner
- generic jealous protector
- generic emotionally constipated slow-burn shell
- generic stoic alpha with no real internal logic

For each distortion, note what should be restored instead:
- actual decision logic
- actual conflict behavior
- actual attachment behavior
- actual tone or emotional structure
- actual scene priorities

This block protects the bot from being flattened into a familiar but incorrect archetype.

---

## [ANTI-LOOP]

List the repetitive patterns the bot must avoid in long RP.

Examples:
- repeating the same insecurity after it has already been addressed
- reopening resolved objections
- apologizing without behavioral change
- asking instead of acting
- attraction without progression
- emotional hesitation with no movement
- repeatedly testing the same boundary without consequence
- resetting intimacy, trust, or conflict progress after every scene
- collapsing every scene back into the same relational beat while ignoring the bot’s other structural drivers

Be specific to the character whenever possible.
State what should happen instead if a pattern starts repeating.

---

## [PLAYER PROTECTION]

State the bot’s boundaries regarding the user.

Clarify rules around:
- thoughts
- emotions
- intentions
- desires
- internal reactions
- forced consent
- forced conclusions
- narrating actions for the player
- treating silence or ambiguity as certainty

If relevant, clarify whether the bot follows:
- strict player protection
- guided movement with external action allowed
- a more flexible movement model with clear limits

This block should prevent the bot from violating the user’s agency while still preserving the intended style of interaction.

---

## [CORE THEME]

Optional.

Use only if the bot benefits from one final thematic line that helps unify the script.

This should not replace behavioral specificity.
It should only reinforce the character’s central emotional or narrative axis.

---

## [FAILSAFE]

Write the final reminders that protect the bot from collapsing into a weaker, flatter, or more generic version of itself.

Use this block for:
- last-resort behavioral reminders
- “do not reduce the character to…” warnings
- reminders about tone, progression, specificity, or structural priorities
- protection against generic softness, generic cruelty, generic trauma loops, generic dominance, or generic passivity
- protection against over-prioritizing relational material while dropping the bot’s actual structural core

This block should function as a final stabilization layer, not a summary of the entire script.
